void Police1_task(void *pvParameters) { Event_t event; while(1) { /* Wait for an event from the police queue */ xQueueReceive(xPoliceQueue, &event, portMAX_DELAY); /* Acquire police resource (semaphore) */ xSemaphoreTake(xPoliceSemaphore, portMAX_DELAY); /* Simulate handling the event (1-3 seconds) */ uint32_t duration_ms = 1000 + rand() % 2000; printf("START: Police-1 handling event ID=[%lu]\n", event.event_id); vTaskDelay(pdMS_TO_TICKS(duration_ms)); printf("COMPLETE: Police-1 finished event ID=[%lu] after %lums\n", event.event_id, duration_ms); /* Send event to logger for tracking */ xQueueSend(xLoggerQueue, &event, portMAX_DELAY); /* Release police resource */ xSemaphoreGive(xPoliceSemaphore); } }

üìÖ Implementation Flow

1. Initialization
Create queues, semaphores, and task handles in main()
2. Event Generation
Event generator creates random emergencies with unique IDs
3. Event Routing
Dispatcher routes events to appropriate department queues
4. Resource Acquisition
Department tasks acquire semaphore before handling events
5. Event Processing
Tasks simulate event handling with random delays (1-3s)
6. Logging & Release
Events logged and resources released for next event

üî® Build & Run Instructions

Prerequisites

  • Linux/POSIX environment
  • GCC compiler
  • Make build tool
  • pthread library

Build Steps

  1. Clone the repository:
    git clone https://github.com/Nadavyehoshua/freeRTOS-Emergency-Dispatch.git
  2. Navigate to the project directory:
    cd freeRTOS-Emergency-Dispatch
  3. Build the project:
    make
  4. Run the simulation:
    ./build/posix_demo
  5. Stop with: Ctrl+C

üîß Technical Details

Task Priorities

  • Priority 4: Dispatcher (highest - must route events quickly)
  • Priority 3: Police, Ambulance, Fire department tasks
  • Priority 2: COVID department tasks
  • Priority 1: Event Generator
  • Priority 0: Logger (lowest)

Resource Limits

  • Police: 3 concurrent events maximum
  • Ambulance: 4 concurrent events maximum
  • Fire: 3 concurrent events maximum
  • COVID: 4 concurrent events maximum

Queue Capacities

  • Dispatcher Queue: 20 events
  • Department Queues: 10 events each
  • Logger Queue: 30 events (largest capacity)
